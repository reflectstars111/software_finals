# 性格雷达·生活指南 — 软件工程答辩文档

## 一、项目概述

**项目名称**：性格雷达·生活指南（Personality Radar · Life Guide）

**一句话定位**：基于大五人格模型与生活方式偏好的综合测评系统，通过科学算法为用户生成个性化餐饮、旅行、社交、穿搭和职业推荐，并支持双人匹配与关系分析。

**目标用户**：大学生及年轻人群，帮助其通过科学测评了解自我，获得可解释的生活决策建议。

**核心闭环**：

```
注册/登录 → 四类测评(性格/饮食/旅游/社交) → 10维度画像报告
    → 个性化推荐(5场景 × 10项) → 用户反馈 → 推荐优化
    → 邀请码匹配 → 双人适配报告
    → 个人中心(历史/反馈/隐私管理)
    → 管理后台(CRUD/统计/日志)
```

## 三、领域模型

### 3.1 10 维度人格+生活画像

系统基于大五人格（OCEAN）扩展为 10 维度模型：

| 分类 | 维度键 | 中文名 | 说明 |
|------|--------|--------|------|
| 人格 | `OPENNESS` | 开放性 | 对新体验、创意和变化的接受程度 |
| 人格 | `CONSCIENTIOUSNESS` | 尽责性 | 对计划、秩序和可靠交付的重视程度 |
| 人格 | `EXTRAVERSION` | 外向性 | 从社交互动中获取能量的倾向 |
| 人格 | `AGREEABLENESS` | 宜人性 | 对关系感受和协调他人的重视程度 |
| 人格 | `NEUROTICISM` | 情绪敏感度 | 对压力和变化的敏感程度（报告反转为情绪稳定性） |
| 生活 | `FOOD_ADVENTURE` | 饮食探索 | 尝试新奇口味和创意餐饮的意愿 |
| 生活 | `FOOD_SOCIAL` | 饮食社交 | 享受聚餐和分享型饮食场景的程度 |
| 生活 | `TRAVEL_ADVENTURE` | 旅行探索 | 向往未知目的地和小众路线的程度 |
| 生活 | `TRAVEL_PLANNING` | 旅行计划 | 对出行确定性和可控性的需求 |
| 生活 | `SOCIAL_ENERGY` | 社交能量 | 主动发起和参与群体活动的能量水平 |

## 四、核心算法

### 4.1 评分引擎（ScoringEngine）

**输入**：用户对每道题的选项选择（李克特5点量表）
**输出**：10维度 0-100 标准化分数

**算法**：
```
对每个维度 d：
  收集该维度在所有题目中的权重值 → weights[d]
  平均值 avg = Σ(weights[d]) / N
  归一化  score = round((avg - 1.0) / 4.0 × 100)
  缺失维度默认 50（中性）
```

每题选项携带对 1-2 个维度的权重映射（如 OPENNESS=5, FOOD_ADVENTURE=5），其余 8 个维度默认为 3（中性）。逆向题取 6-score 反转。

**特点**：纯函数、无副作用、完全可单元测试。支持 10 维度同时计算，处理缺失维度。

### 4.2 推荐排序引擎（RecommendationRanker）

**设计原则**：性格画像为主信号（68%），推荐项品质为辅（22%），反馈和规则为微调。

**公式**：
```
score = profileFit × 0.68        ← 性格匹配（主信号）
      + baseScore × 0.22         ← 基础品质
      + preferenceDelta           ← 反馈微调（tanh限幅 ±10）
      + ruleDelta                 ← 规则微调（clamp限幅 ±6）
```

**性格匹配度**（profileFit）按标签计算加权维度平均：

| 标签 | 关联维度（权重分配） |
|------|---------------------|
| `explore` | OPENNESS(0.45) + FOOD_ADVENTURE(0.25) + TRAVEL_ADVENTURE(0.30) |
| `structured` | CONSCIENTIOUSNESS(0.65) + TRAVEL_PLANNING(0.25) + EMOTIONAL_STABILITY(0.10) |
| `social` | EXTRAVERSION(0.50) + SOCIAL_ENERGY(0.30) + FOOD_SOCIAL(0.15) + AGREEABLENESS(0.05) |
| `gentle` | NEUROTICISM(0.45) + AGREEABLENESS(0.25) + INTROVERSION(0.20) + TRAVEL_PLANNING(0.10) |

维度分数使用**偏差放大**（`50 + (raw - 50) × 1.6`），使高低分用户产生明显差异。

**反馈微调**：
```
preferenceDelta = tanh(preference / 18) × 10
```
- 偏好范围 [-30, 30]，tanh 函数确保微调有界
- LIKE +3 / NEUTRAL +1 / DISLIKE -3（每次反馈的增量）

**得分聚合**：多标签推荐项取各标签得分的平均值，最终分数四舍五入到 0-100 整数。

### 4.3 匹配引擎（MatchEngine）

**输入**：两个用户各自最新的 PERSONALITY 测试结果（10维分数）
**输出**：0-100 契合度分数

**算法**：
```
对 10 个维度分别计算：diff[d] = |scoreA[d] - scoreB[d]|
平均差异 = Σ(diff) / 10
契合度 = max(0, 100 - 平均差异)
```

**文本解释生成**：
- 按维度差异升序排列，前 2 个为"高契合维度"（strengths）
- 后 2 个为"差异较大维度"（warnings）
- 根据总体分数生成摘要（≥80 高度契合 / ≥60 整体匹配 / <60 差异明显）

### 4.4 报告聚合

报告不单独使用 PERSONALITY 测试——而是**聚合全部 4 类测试**的最新结果：对每个维度取各测试类型的最高分（`Math.max`），合并为完整的 10 维度画像。这样生活方式维度（如 FOOD_ADVENTURE）能从对应的饮食偏好测试中获得有效分数。

## 五、安全设计

| 层面 | 机制 |
|------|------|
| 认证 | JWT (HMAC-SHA384)，24h 过期，JWT_SECRET 启动时由 CSPRNG 自动生成 48 字节 |
| 密码 | BCrypt 编码，5 次失败锁定 10 分钟 |
| 授权 | Spring Security 角色控制（USER/ADMIN），ROLE_ADMIN 保护 /api/admin/** |
| 数据隔离 | 所有查询通过 `findByOwner(user)` / `findByUser(user)` 限定当前用户 |
| 并发 | 注册用 DB UNIQUE 约束兜底（无 TOCTOU），DataIntegrityViolationException → 409 |
| 限流 | RateLimitFilter：auth/share/admin 端点每 IP 每分钟 10 次 |
| 隐私 | 分享报告只展示维度，不包含手机号；匹配报告不暴露对方完整答题 |
| CORS | 白名单来源配置，生产用具体域名 |

## 九、技术亮点

1. **10 维度科学模型**：在大五人格基础上扩展生活方式维度，每题选项携带双重权重映射
2. **可解释推荐**：68% 权重来自性格匹配，每个推荐项的得分可追溯到具体维度：你的开放性偏高 → explore 标签得分 +X
3. **反馈闭环**：tanh 函数确保单次反馈不会主导排序，长期反馈逐步塑造推荐
4. **报告聚合**：不是简单取人格测试结果，而是跨 4 类测试取 max 合并，让饮食偏好测试真正影响饮食推荐
5. **去 AI 味设计**：Radar Chromatic 专属视觉系统，Canvas 动画雷达扫掠，拒绝 Inter/紫色渐变/奶油色三件套
6. **并发安全**：注册 TOCTOU 消除，DB 约束兜底，DataIntegrityViolationException 全局处理
7. **TypeScript 类型统一**：前后端共享 10 维度枚举键，单一 types.ts 来源
8. **Docker 一键部署**：`reactivate-default-users.ps1` 脚本完整编排构建→部署→验证→公网隧道
