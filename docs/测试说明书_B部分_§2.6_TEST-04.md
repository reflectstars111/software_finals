### 2.6.4 测试设计

**a) 控制方式**：半自动测试。API 使用 curl 验证算法正确性（得分范围、排序、反馈闭环），前端使用 Chrome DevTools 验证 UI 渲染和交互反馈。

**b) 输入数据选择说明**：

| 数据类别 | 选择理由 |
|---------|---------|
| 5 个有效场景（food/travel/social/outfit/career）| 验证全部场景推荐可用 |
| 无效场景（invalid）| 验证枚举校验 |
| 无测试记录的用户 | 验证前置条件检查（400 错误） |
| LIKE/NEUTRAL/DISLIKE 三种反馈 | 验证反馈闭环（权重增量 +3/+1/-3） |
| 不存在的推荐项 ID（9999）| 验证 404 处理 |
| 用户 13900000001 的高社交能量画像 | 验证 social 标签推荐项排序靠前 |

**c) 预期输出**：
- 推荐排序算法公式：

  ```
  score = profileFit × 0.68       // 性格匹配（主信号，含偏差放大 ×1.6）
        + baseScore × 0.22        // 基础品质
        + preferenceDelta         // 反馈微调（tanh(pref/18) × 10，限幅 ±10）
        + ruleDelta               // 规则微调（weight × 1.2，限幅 ±6）
  ```

- 性格匹配按标签加权平均：`explore`(OPENNESS×0.45+FOOD_ADVENTURE×0.25+TRAVEL_ADVENTURE×0.30) / `structured`(CONSCIENTIOUSNESS×0.65+TRAVEL_PLANNING×0.25+EMOTIONAL_STABILITY×0.10) / `social`(EXTRAVERSION×0.50+SOCIAL_ENERGY×0.30+FOOD_SOCIAL×0.15+AGREEABLENESS×0.05) / `gentle`(NEUROTICISM×0.45+AGREEABLENESS×0.25+INTROVERSION×0.20+TRAVEL_PLANNING×0.10)
- 每场景返回 ≤10 项，按 score 降序
- 反馈不影响当次排序，下次加载时生效
- 偏好权重范围 [-30, 30]，使用 tanh 函数确保微调有界
- 所有失败响应 code≠0，含中文提示
