# 性格雷达·生活指南完整初版实施计划

## Summary

基于 [需求文档.md](D:/软工/大作业/需求文档.md)，从零搭建完整前后端分离软件。当前目录只有需求文档，没有现成代码；初版按用户选择的“完整需求版”规划，采用“Docker 官方栈”落地：Vue3 前端、Spring Boot 后端、MySQL 8.0、Redis、Docker Compose。

目标交付一个可运行、可演示、可测试的完整闭环版本，覆盖注册登录、测试答题、画像报告、雷达图、推荐、反馈优化、双人适配、分享、后台管理、权限隔离和基础测试。

## Key Changes

- 创建单仓工程结构：
  - `frontend/`：Vue3 + Vite + TypeScript + Vue Router + Pinia + Axios + ECharts。
  - `backend/`：Spring Boot 3.x + JDK17 + Spring Security + JWT + Spring Data JPA + MySQL + Redis。
  - `infra/`：Docker Compose、MySQL 初始化、Redis、后端容器运行配置。
  - `docs/`：接口说明、部署说明、测试说明。
- Docker Compose 提供 MySQL 8.0、Redis 7、后端 JDK17/Maven 运行环境，规避本机 Java 1.8、缺 Maven、缺 MySQL 的问题。
- 使用统一 API 响应格式：
  - `ApiResponse<T> { code, message, data }`
  - 成功：`code = 0`
  - 失败：返回业务错误码和友好中文提示，不暴露异常堆栈。
- 使用 JWT 登录认证和角色权限：
  - 普通用户角色：`USER`
  - 管理员角色：`ADMIN`
  - 默认种子管理员：手机号 `13800000000`，密码 `Admin@123456`
- 数据模型覆盖：
  - 用户、题目、选项、测试结果、推荐项、推荐反馈、适配报告、分享链接、后台操作日志。
- 初版内置种子数据：
  - 基础性格、饮食偏好、旅游偏好三类题目。
  - 大五人格维度：开放性、尽责性、外向性、宜人性、神经质。
  - 餐饮、旅游、穿搭、生涯规划推荐项。
  - 推荐标签与权重规则。

## Implementation Plan

1. 搭建工程与基础设施  
   创建 `frontend/`、`backend/`、`infra/`，配置 Docker Compose，包含 `mysql:8.0`、`redis:7-alpine` 和后端 JDK17/Maven 容器。后端通过容器运行，前端用本机 Node/npm 开发。

2. 实现后端基础框架  
   建立 Spring Boot 项目，配置 MySQL、Redis、JPA、统一异常处理、统一响应结构、参数校验、跨域配置、日志配置和环境变量配置。

3. 实现认证与权限  
   实现手机号注册、密码 BCrypt 加密、登录签发 JWT、当前用户查询、资料修改、角色鉴权、管理员访问控制和越权拦截。

4. 实现题库与测试模块  
   管理三类测试题库，用户获取题目、提交答案、后端按选项权重计算维度得分，保存答题记录和 `TestResult`。得分范围为 0-100，整数保留。

5. 实现画像报告与雷达图数据  
   后端提供最新报告接口，输出维度分数、文字解析、场景建议和 ECharts 雷达图所需数据。前端报告页渲染雷达图、分区报告和历史记录。

6. 实现场景化推荐  
   后端根据最新测试结果、推荐项标签、基础权重和用户反馈权重计算推荐列表。前端提供餐饮、旅游、穿搭、生涯规划推荐视图。

7. 实现反馈优化  
   用户对推荐项提交“喜欢 / 一般 / 不喜欢”和 0-100 字文字反馈。后端保存反馈并调整用户对应场景标签权重，后续推荐结果受反馈影响。

8. 实现双人适配  
   用户通过好友 ID 发起适配。后端复用双方最新测试结果，按维度差值计算 0-100 适配分，保留 1 位小数，并生成互补优势、分歧预警和相处建议。

9. 实现分享能力  
   前端支持将报告区域导出为 PNG；后端生成加密分享 token 和短链接记录。初版分享链接展示只读报告，不允许修改数据。

10. 实现后台管理  
   管理员可维护题目、选项、推荐项和推荐规则，可查看用户反馈、基础统计、操作日志。普通用户访问后台接口和页面时必须被拒绝。

11. 完成前端页面  
   页面包括登录/注册、首页、测试列表、答题页、综合报告页、推荐页、双人适配页、个人中心、后台管理页。界面移动端优先，桌面端自适应。

12. 编写部署与使用文档  
   文档说明 Docker 启动、前端启动、默认账号、接口地址、数据库初始化、测试命令和常见问题。

## Public APIs

- `POST /api/auth/register`：手机号注册。
- `POST /api/auth/login`：登录并返回 JWT。
- `GET /api/users/me`：获取当前用户资料。
- `PUT /api/users/me`：修改当前用户资料。
- `GET /api/questions?type=personality|food|travel`：获取测试题目。
- `POST /api/tests/submit`：提交测试答案并生成测试结果。
- `GET /api/tests/history`：查看历史测试记录。
- `GET /api/reports/me`：获取当前用户综合画像报告。
- `GET /api/recommendations?scene=food|travel|outfit|career`：获取推荐列表。
- `POST /api/recommendations/{id}/feedback`：提交推荐反馈。
- `POST /api/matches`：发起双人适配。
- `GET /api/matches/{id}`：查看适配报告。
- `POST /api/shares/report`：生成报告分享链接。
- `GET /api/shares/{token}`：查看只读分享报告。
- `GET/POST/PUT/DELETE /api/admin/questions`：后台题库维护。
- `GET/POST/PUT/DELETE /api/admin/recommendation-items`：后台推荐项维护。
- `GET /api/admin/feedback`：后台反馈查看。
- `GET /api/admin/logs`：后台日志查看。
- `GET /api/admin/stats`：后台基础统计。

  - 性格得分计算。
  - 推荐权重计算。
  - 双人适配分数与预警生成。
  - JWT 鉴权和角色权限。
- 后端集成测试：
  - 注册登录完整流程。
  - 测试提交到报告生成。
  - 推荐反馈后推荐排序变化。
  - 普通用户访问后台接口返回拒绝。
  - 好友 ID 不存在或好友未测试时返回友好错误。
- 前端测试：
  - 登录、测试、报告、推荐、适配、后台页面基本渲染。
  - 移动端宽度下无严重遮挡、错位和横向滚动。
  - 雷达图在报告页正常渲染。
- 端到端测试：
  - 用户注册 -> 完成测试 -> 查看报告 -> 获取推荐 -> 提交反馈。
  - 两个用户完成测试 -> 发起双人适配 -> 查看适配报告。
  - 管理员登录 -> 修改题库或推荐项 -> 用户端可见变化。
- 性能与验收：
  - 常规接口响应目标不超过 2 秒。
  - 测试提交、报告生成、推荐生成、适配生成目标不超过 3 秒。
  - 提供 JMeter 脚本验证 100 用户并发访问主要接口。

## Assumptions

- 按“完整需求版”规划，但初版仍以课程演示可运行为第一优先级。
- 第三方登录、支付、广告、即时通讯、复杂机器学习推荐算法不实现。
- 分享短链接在本系统内可访问，不接入真实外部社交平台。
- 推荐算法采用规则引擎、标签匹配和反馈权重修正，不训练模型。
- 数据加密初版采用密码 BCrypt、敏感字段必要脱敏展示；生产级密钥管理后续扩展。
- 数据库每日备份在初版中通过 Docker/脚本方案提供，不做复杂灾备平台。
- 后续退出 Plan Mode 后，才能开始创建工程文件、安装依赖和实现软件。
