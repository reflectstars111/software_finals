# 个性化社区（PersonaFeed）— 需求文档

> 状态：✅ 已确认，可进入开发
> 归属：推荐模块的子功能
> 目标：在推荐模块中加入 UGC 社区，用户发布图文帖 → AI 为帖子生成人格向量 → 基于人格契合度推荐帖子

---

## 一、核心概念

### 1.1 帖子人格向量（Post Personality Vector）

每篇帖子拥有一个 **与用户大五人格完全对应的 10 维向量**，由 AI 根据帖子图文内容自动生成：

| 维度              | 含义（帖子侧）                |
| ----------------- | ----------------------------- |
| OPENNESS          | 帖子体现的"开放性/新奇感"程度 |
| CONSCIENTIOUSNESS | 帖子体现的"计划性/条理感"程度 |
| EXTRAVERSION      | 帖子体现的"社交热闹感"程度    |
| AGREEABLENESS     | 帖子体现的"温和治愈感"程度    |
| NEUROTICISM       | 帖子体现的"情绪表达强度"      |
| FOOD_ADVENTURE    | 饮食领域的探索程度            |
| FOOD_SOCIAL       | 饮食领域的社交属性            |
| TRAVEL_ADVENTURE  | 旅行领域的探索程度            |
| TRAVEL_PLANNING   | 旅行领域的计划属性            |
| SOCIAL_ENERGY     | 社交能量强度                  |

向量的来源有两个：

1. **AI 初始赋值**：发帖时由 AI 分析图文内容生成
2. **社群反馈修正**：帖子向量 = 所有点赞用户的 10 维人格向量的**算术平均**（见 §4）

### 1.2 关键定义补充

#### 1.2.1 活跃帖子

发布30天内、近7天至少1次点赞/评论/收藏的帖子；超过30天无互动标记为非活跃，不参与契合度推荐排序，仅可在最新列表检索。

#### 1.2.2 契合度展示规则

- 登录用户浏览社区：卡片标签文字 `你的契合度：XX%`；
- 访客/未完成性格测试用户：隐藏契合度标签；
- 帖子详情页统一说明：该数值基于你的性格画像与帖子内容匹配得出。

## 二、用户故事

### 2.1 发帖（U1）

> **前置条件：** 用户必须已完成性格测试（PERSONALITY）。未测试者点击发帖会弹出提示"请先完成性格测试再发布动态"。

1. 用户点击"发布动态"
2. 上传 0–9 张图片（可选，纯文字也可）
3. 输入正文（1–2000 字）
4. 选择 **领域标签**（必选，单选）：饮食 / 旅游 / 穿搭 / 生涯 / 社交 / 其他
5. 选择 **风格标签**（可选，多选）：
   - 约束：同一领域下最多选择3个风格标签；自定义标签长度1–15字符，需通过敏感词过滤
   - 饮食类：轻食·健康 / 咸辣重口 / 甜品烘焙 / 探店打卡 / 家常料理 / 深夜食堂
   - 旅游类：自然风光 / 城市漫游 / 文化古迹 / 冒险极限 / 躺平度假
   - 穿搭类：简约通勤 / 复古文艺 / 潮流街头 / 运动休闲 / 日系森系
   - 生涯类：技术成长 / 创业日记 / 斜槓日常 / 职场吐槽
   - 社交类：聚会活动 / 深度交流 / 独处时光
   - 其他：自定标签或通用
6. （可选）查看 AI 智能标签建议，可采纳或忽略
7. 点击"提交审核"
8. 提交后进入「审核中」状态，页面锁定不可编辑/重复提交，展示“审核中，请稍候”文案

### 2.2 AI 审核（U1 续）

提交后：

1. AI 对图文进行**内容安全审核**（MVP阶段仅文本审核；二期图片单独AI审核）
2. AI 同时生成帖子的 **10 维人格向量**（数值自动 clamp(0,100) 并强制转为整数）
3. 审核通过 → 帖子发布，进入社区广场
4. 审核不通过 → 告知原因，可修改重发
5. AI 服务异常降级：AI 超时/报错时自动切换为 Mock 模式，不阻断发帖流程，后台记录异常日志告警

> Mock 模式下：
>
> - 审核永远返回"通过"
> - 初始向量为基于内容 hash 的伪随机值
> - 点赞更新逻辑仍执行算术平均算法，保证流程可完整自测

### 2.3 浏览社区（U2）

1. 用户进入推荐模块 → 通过**"进入社区"按钮**打开社区子页面
2. 顶部下拉框切换排序方式：**"推荐"（人格契合度） / "最新"**
3. 未完成性格测试/清空测试记录的用户：只能看"最新"排序，不能看"推荐"
4. 页面展示帖子列表（卡片列表）：
   - 分页规则：20条/页，下拉无限滚动加载
   - 空状态：无帖子/筛选后无结果时展示“暂无相关内容”文案+插图
5. 每篇帖子卡片显示：
   - 发布者头像/昵称（脱敏展示，仅显示公开昵称，隐藏用户ID）
   - 正文摘要（前 120 字）
   - 领域标签 + 风格标签
   - 点赞数 / 评论数 / 收藏数
   - 契合度标签（登录用户显示「你的契合度：XX%」，访客/未测试用户隐藏）
6. 点击卡片进入帖子详情页

### 2.4 帖子详情 & 互动（U3）

1. 查看完整图文
2. **点赞**—— 点击点赞，再点取消；点赞/取消操作触发防抖处理（10s内重复操作合并为一次），向量重算为异步定时任务（延迟5分钟批量重算）
3. **收藏**—— 个人收藏夹
4. **评论**—— 一级评论即可（先不做嵌套回复）：
   - 发表时经**敏感词过滤**
   - 分页规则：10条/页，下拉加载更多
   - 权限：帖主可删全部评论、评论作者仅能删自己、管理员可删所有
   - 举报：支持评论举报功能，流程同帖子举报
5. **举报**—— 违规内容举报
6. **分享**—— 生成图文分享海报/复制短链接；外链免登录查看（仅阅读，无互动按钮）

### 2.5 我的帖子（U4）

1. 个人中心新增"我的帖子"入口
2. 查看自己发布的所有帖子（分页加载，20条/页）
3. 可**编辑**（重新触发 AI 审核 + 向量生成）/ **删除**（软删除，active=0）自己的帖子
4. 查看每篇帖子的互动数据（点赞数、收藏数、评论数）
5. 空状态：无发布帖子时展示“你还未发布任何动态”文案+引导发帖按钮
6. 违规下架帖子：管理员下架的帖子仅作者可见，标记“已下架”标签

### 2.6 收藏夹（U5）

1. 个人中心新增"我的收藏"入口
2. 按领域筛选收藏的帖子
3. 支持批量取消收藏
4. 空状态：无收藏帖子时展示“你还未收藏任何动态”文案

### 2.7 帖子分享（U6）

1. 帖子详情页点击"分享"按钮，支持生成图文分享海报、复制短链接
2. 外链免登录查看帖子，仅展示内容，隐藏互动按钮（点赞/收藏/评论/举报）
3. 外链页面隐藏契合度标签

## 三、AI 接入点总结

| 环节     | AI 能力                               | 输入               | 输出                                    | 优先级      |
| -------- | ------------------------------------- | ------------------ | --------------------------------------- | ----------- |
| 发帖审核 | 内容安全审核（MVP仅文本，二期加图片） | 文本（+ 二期图片） | 通过/不通过 + 原因                      | MVP（mock） |
| 向量生成 | 帖子人格向量推断                      | 文本               | 10 维得分 [0-100]（自动clamp+强制整数） | MVP（mock） |
| 智能标签 | 自动标签建议                          | 文本               | 推荐 3–5 个标签（用户可选）            | 第二期      |

> AI 异常降级：AI 服务不可用时自动切换 Mock 模式，后台记录异常日志告警，不阻断核心流程。
> Mock 模式：审核永远通过，初始向量基于内容 hash 伪随机生成，点赞更新逻辑正常执行。

### 3.1 评论敏感词过滤

- **不使用 AI**，使用本地敏感词库（正则匹配）
- 命中敏感词 → 拒绝发布，提示"评论含不当内容，请修改"
- 敏感词库维护在配置文件中，可随时增补

## 四、推荐算法详细设计

### 4.1 帖子向量计算

帖子发布时，AI 生成初始向量，后续由社群反馈决定：

**规则：帖子向量 = 所有点赞用户的 10 维人格向量的算术平均**

$$
V_{post} = \frac{1}{n} \sum_{k=1}^{n} V_{user_k}
$$

其中 $n$ 为点赞人数，$V_{user_k}$ 为第 $k$ 个点赞用户的 10 维向量（从 `t_test_result` 最新测试记录读取）。

| 场景              | 帖子向量                                                     |
| ----------------- | ------------------------------------------------------------ |
| 刚发布，无人点赞  | 使用 AI 初始向量                                             |
| 有人点赞/取消点赞 | 防抖处理（10s内重复操作合并），延迟5分钟异步批量重算         |
| 帖子被编辑        | AI 重新生成初始向量，已有赞继续参与平均                      |
| 点赞数>200        | 改用「初始AI向量 + 前200点赞用户均值缓存」，平衡性能与准确度 |

**性能优化：**

1. 向量计算改为异步定时任务，避免实时计算阻塞接口；
2. 帖子ai_vector字段增加Redis缓存，缓存有效期1小时；
3. 点赞数>200时启用降级策略，不再全量计算所有点赞用户均值。

**直觉解释：**

- 如果"高开放性+高外向性"的用户群体喜欢某帖子 → 帖子向量自然偏向高开放性+高外向性
- 帖子向量天然收敛到"喜欢它的人群"的平均画像
- 算法简单、可解释、无累积漂移问题

### 4.2 契合度计算

复用 `MatchEngine` 并做兼容改造：

1. 原有5维双人匹配逻辑保持不变；
2. 新增10维数组入参的重载方法，计算公式与原有完全一致：

$$
similarity(U, P) = 100 - \frac{1}{10} \sum_{i=1}^{10} |U_i - P_i|
$$

### 4.3 推荐排序

1. 筛选所有**活跃帖子**；
2. 计算当前用户与活跃帖子的契合度；
3. 按契合度降序排列，同分数下按7天互动总量（点赞+评论+收藏）降序；
4. 取 Top 10；
5. 从当前页第11–30名中随机抽取 1–2 篇混入（每页都保证多样性）；
6. 返回混合后的列表（20条/页，无限滚动加载）。

### 4.4 "最新"排序

按 `created_at` 降序，20条/页，无限滚动加载。未完成性格测试/清空测试记录的用户仅能看到此排序。

3. 软删除：active=0，仅作者可见；
4. 管理员下架：active=1 且 新增字段 `is_down`=1，仅作者可见并标记“已下架”。

## 六、API 设计

### 6.1 帖子 CRUD

| 方法       | 路径                           | 说明                                                               |
| ---------- | ------------------------------ | ------------------------------------------------------------------ |
| `POST`   | `/api/posts`                 | 发布帖子（含图文 + 标签，触发 AI 审核）                            |
| `GET`    | `/api/posts`                 | 社区广场（按人格契合度/最新排序，支持 ?domain=FOOD 筛选，20条/页） |
| `GET`    | `/api/posts/:id`             | 帖子详情                                                           |
| `PUT`    | `/api/posts/:id`             | 编辑帖子（仅作者，重新触发 AI 审核）                               |
| `DELETE` | `/api/posts/:id`             | 删除帖子（仅作者，软删除）                                         |
| `GET`    | `/api/posts/mine`            | 我的帖子列表（分页，20条/页）                                      |
| `GET`    | `/api/posts/share/:shortUrl` | 帖子分享外链（免登录，仅阅读）                                     |

### 6.2 互动

| 方法       | 路径                           | 说明                          |
| ---------- | ------------------------------ | ----------------------------- |
| `POST`   | `/api/posts/:id/like`        | 点赞（防抖+异步重算向量）     |
| `DELETE` | `/api/posts/:id/like`        | 取消点赞（防抖+异步重算向量） |
| `POST`   | `/api/posts/:id/favorite`    | 收藏                          |
| `DELETE` | `/api/posts/:id/favorite`    | 取消收藏                      |
| `DELETE` | `/api/posts/favorites/batch` | 批量取消收藏                  |
| `GET`    | `/api/posts/favorites`       | 我的收藏列表（分页，20条/页） |

### 6.3 评论

| 方法       | 路径                                          | 说明                             |
| ---------- | --------------------------------------------- | -------------------------------- |
| `POST`   | `/api/posts/:id/comments`                   | 发表评论（敏感词过滤）           |
| `GET`    | `/api/posts/:id/comments`                   | 查看评论列表（分页，10条/页）    |
| `DELETE` | `/api/posts/:id/comments/:commentId`        | 删除评论（帖主/评论作者/管理员） |
| `POST`   | `/api/posts/:id/comments/:commentId/report` | 举报评论                         |

### 6.4 举报

| 方法     | 路径                      | 说明     |
| -------- | ------------------------- | -------- |
| `POST` | `/api/posts/:id/report` | 举报帖子 |

### 6.5 管理端接口（二期）

| 方法    | 路径                          | 说明                                   |
| ------- | ----------------------------- | -------------------------------------- |
| `GET` | `/api/admin/posts/stats`    | 帖子数据统计（总数量、各领域发帖量等） |
| `PUT` | `/api/admin/posts/:id/down` | 下架帖子（管理员）                     |
| `GET` | `/api/admin/posts/reports`  | 举报列表                               |

### 6.6 统一规范

1. 所有分页接口返回格式：`{code: 200, data: {list: [], total: 0}, msg: "success"}`；
2. 错误码定义：
   - 400：参数错误/敏感词命中
   - 401：未登录/权限不足
   - 403：未完成性格测试
   - 500：服务器异常/AI服务超时
   - 503：AI服务不可用（已降级为Mock）

```
├── 点赞/收藏/分享 按钮 + 契合度标签（登录用户可见）
├── 评论列表（分页，10条/页）
│   └── CommentItem.vue (新增)
│       ├── 评论内容
│       ├── 举报按钮
│       └── 删除按钮（权限控制）
├── 评论输入框（敏感词过滤提示）
└── 空状态：无评论（文案+插图）

PostShareView.vue (新增)
├── 帖子纯展示内容（无互动按钮）
└── 分享引导文案
```

### 7.4 交互补充规范

1. 加载态：所有列表/提交操作展示骨架屏/加载动画，文案“加载中...”；
2. 失败态：接口报错展示“加载失败，请重试”+ 刷新按钮；
3. 审核中态：发帖提交后展示“审核中，预计5分钟内完成”，页面不可编辑；
4. 标签联动：切换领域标签时，清空已选风格标签并加载对应领域的风格标签；
5. 空状态全覆盖：我的帖子/收藏/评论/筛选结果为空时均展示对应文案+插图。

## 十、风险与合规补充

### 10.1 数据隐私风险

- 所有用户信息仅展示脱敏昵称，不泄露手机号/ID等敏感数据；
- 帖子内容存储符合UGC合规要求，支持违规下架/删除。

### 10.2 性能风险

- 向量计算异步化+缓存，避免高并发下接口超时；
- 点赞数>200时启用降级策略，平衡性能与准确度。

### 10.3 内容合规风险

- 敏感词过滤覆盖帖子正文/评论/自定义标签；
- 管理员可下架违规内容，建立账号处罚梯度，避免违规UGC扩散。

### 10.4 配置管理

- 敏感词库、AI服务地址、向量缓存时长等配置化管理，支持动态调整；
- 新增社区模块风险项监控（违规率、接口超时率、AI异常率）。
