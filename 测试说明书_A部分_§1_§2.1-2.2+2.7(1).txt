§1 引言
1.1 编写目的
本文档为”人格画像雷达系统”的测试说明书，用于：
规定本项目的测试范围、测试项、测试方法、进度安排和资源需求；
记录测试执行结果、发现的缺陷及对软件功能的结论；

1.2 背景
课程项目背景：
本项目为武汉大学2024级大二软件工程课程设计项目，由6人团队（编号 A~F）协作完成。项目总开发周期约14周，涵盖需求分析、系统设计、编码实现、测试执行与文档交付全流程。本测试说明书对应项目”第四部分”，依据 GB/T 8567-1988 标准，融合”测试计划”与”测试分析报告”两份模板编制。
项目概述：
“人格画像雷达系统”是一套基于大五人格理论的轻量级 Web 应用，面向 18-35 岁年轻群体，提供多维性格测评、雷达图可视化、生活场景个性化推荐、双人性格适配与报告分享等核心能力。系统采用前后端分离架构：前端使用 Vue 3 + Vite + TypeScript + ECharts，后端使用 Spring Boot 3.x + JDK 17，数据库采用 MySQL 8.0（Docker 容器化部署），缓存使用 Redis 7，通过 Docker Compose 统一管理基础设施。系统覆盖 10 个评测维度（5 个人格维度 + 5 个生活方式维度），支持基础性格、饮食偏好、旅游偏好、社交倾向四类测试，可输出餐饮、旅游、社交、穿搭、生涯规划五类场景推荐。管理员可通过后台维护题库、推荐规则与查看运营数据。

1.3 定义
术语/缩写
说明
大五人格
心理学中描述人格的五大维度模型：开放性(O)、尽责性(C)、外向性(E)、宜人性(A)、情绪敏感度(N)。本项目在报告中将其反转为”情绪稳定性”以提升用户理解度
雷达图
以五边形或多边形可视化展示用户多维度性格得分的图表，本项目使用 ECharts 渲染
推荐引擎
根据用户性格得分与推荐项标签权重的匹配算法，输出个性化的生活场景推荐
JWT
JSON Web Token，用于用户登录后的身份认证与角色鉴权
适配分
基于双方各维度差值计算的 0-100 兼容性指标，以百分比形式展示（如 85.2%）

§2 测试计划与设计
2.1 软件说明
本系统的功能模块如下：
人格画像雷达系统├── 1. 用户认证模块│   ├── 手机号注册、密码登录│   ├── JWT 身份认证│   └── 角色鉴权（USER / ADMIN）├── 2. 题库与测试模块│   ├── 基础性格测试、饮食偏好测试、旅游偏好测试、社交倾向测试│   └── 答案提交与维度得分计算（0-100）├── 3. 画像报告模块│   ├── 综合画像报告生成│   ├── 雷达图数据输出│   └── 报告快照历史├── 4. 场景推荐模块│   ├── 餐饮、旅游、社交、穿搭、生涯规划推荐│   └── 用户反馈（喜欢/一般/不喜欢）├── 5. 双人适配模块│   ├── 适配分计算（0-100）│   ├── 手机号匹配 + 邀请码匹配（90天有效）│   └── 互补优势、分歧预警、相处建议├── 6. 分享模块│   ├── 报告分享 token 生成│   ├── 只读分享访问│   └── 分享链接管理（列表/撤销）└── 7. 后台管理模块    ├── 题库维护、推荐项维护    ├── 推荐标签规则维护    ├── 用户管理、反馈查看    └── 操作日志与基础统计
输入简述：用户通过前端页面提交手机号/密码、测试答案、推荐反馈、好友手机号等数据。
输出简述：系统返回 JWT token、维度得分（0-100）、雷达图数据、推荐列表、适配分（0-100，1 位小数）、分享链接等。
